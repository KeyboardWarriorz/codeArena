서블릿 매핑(Servlet mapping)은 Java 웹 애플리케이션에서 특정 URL 패턴과 서블릿 클래스를 연결하는 설정입니다. 클라이언트로부터 들어오는 요청이 특정 URL 패턴과 일치할 때, 해당 서블릿 클래스가 실행되어 요청을 처리합니다.

서블릿 매핑은 주로 웹 애플리케이션의 구성 설정 파일인 web.xml에서 수행됩니다. web.xml 파일에 서블릿 매핑을 추가하면, 해당 URL 패턴으로 들어오는 요청이 매핑된 서블릿 클래스로 전달됩니다.

📒

서블릿 매핑의 예시

<!-- web.xml 파일의 일부분 -->
        <servlet>
            <servlet-name>MyServlet</servlet-name>
            <servlet-class>com.example.MyServlet</servlet-class>
        </servlet>
    
        <servlet-mapping>
            <servlet-name>MyServlet</servlet-name>
            <url-pattern>/myservlet/*</url-pattern>
        </servlet-mapping>

▲ 위의 예시에서는 MyServlet이라는 서블릿 클래스를 /myservlet/* URL 패턴에 매핑하고 있습니다.

  • 클라이언트가 /myservlet/somepage URL로 요청을 보냅니다.
  • 웹 서버(서블릿 컨테이너)는 해당 요청이 /myservlet/* 패턴에 일치한다는 것을 확인합니다.
  • MyServlet 클래스의 인스턴스를 생성하여 요청을 처리합니다.

예시에서 /myservlet/* 패턴은 /myservlet/로 시작하는 모든 URL과 일치하며, 그 뒤에 다른 경로나 쿼리 문자열이 올 수 있습니다.

예를 들어, /myservlet/page1, /myservlet/subfolder/page2, /myservlet?action=doSomething 등의 URL이 일치할 수 있습니다.

👩‍🏫

서블릿 매핑을 설정함으로써 특정 URL 패턴과 서블릿 클래스를 연결할 수 있으므로, 클라이언트의 요청을 해당 서블릿으로 라우팅하여 웹 애플리케이션의 동적인 처리를 가능하게 합니다.
