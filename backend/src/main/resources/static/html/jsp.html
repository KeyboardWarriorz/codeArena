  • HTML이란?

    HTML

    • HTML?
      • Hyper Text Markup Language
      • 웹 페이지를 작성(구조화)하기 위한 언어
    • Hyper Text
      • 참조(하이퍼링크)를 통해 사용자가 한 문서에서 다른 문서로 즉시 접근할 수 있는 텍스트
      • 위키 시스템과 유사!
    • Markup Language
      • 태그 등을 이용하여 문서나 데이터의 구조를 명시하는 언어
      • 대표적인 예 : HTML, Markdown
    • .html
      • HTML 파일
  • HTML 기본 구조

    HTML 기본구조

    • html : 문서의 최상위(root) 요소
    • head : 문서 메타데이터 요소
      • 메타데이터란? - 데이터를 설명해주는 데이터 (ex. 사진을 찍으면 확인할 수 있는 날짜, 조리개값, 노출 등)
      • 문서 제목, 인코딩, 스타일, 외부 파일 로딩 등
      • 일반적으로 브라우저에 나타나지 않는 내용
    • body : 문서 본문 요소
      • 실제 화면 구성과 관련된 내용
    • head 예시
      • < title > : 브라우저 상단 타이틀
      • < meta > : 문서 레벨 메타데이터 요소
      • < link > : 외부 리소스 연결 요소 (CSS 파일, favicon 등)
      • < script > : 스크립트 요소 (JavaScript 파일/코드)
      • < style > : CSS 직접 작성
    • head 예시 : Open Graph Protocol
      • 메타 데이터를 표현하는 새로운 규약
        • HTML 문서의 메타 데이터를 통해 문서의 정보를 전달
        • 메타정보에 해당하는 제목, 설명 등을 쓸 수 있도록 정의
  • DOM 트리

    DOM(Document Object Model) 트리

    • 텍스트 파일인 HTML 문서를 브라우저에서 렌더링 하기 위한 구조
      • HTML 문서에 대한 모델을 구성함
      • HTML 문서 내의 각 요소에 접근 / 수정에 필요한 프로퍼티와 메서드를 제공함
  • HTML element (요소)

    요소(element)

    • HTML의 요소는 태그와 내용(contents)로 구성되어 있다.
    • HTML 요소는 시작 태그와 종료 태그 그리고 태그 사이에 위치한 내용으로 구성
      • 태그(Element, 요소)는 컨텐츠(내용)를 감싸는 것으로 그 정보의 성격과 의미를 정의
    • 내용이 없는 태그들
      • br(break line), hr(horizental rule), img, input, link, meta
    • 요소는 중첩(nested) 될 수 있음
      • 요소의 중첩을 통해 하나의 문서를 구조화
      • 여는 태그와 닫는 태그의 쌍을 잘 확인해야 함
        • 오류를 반환하는 것이 아닌 그냥 레이아웃이 깨진 상태로 출력되기 때문에, 디버깅이 힘들어 질 수 있음
  • HTML attribute (속성)

    속성(attribute)

    • 태그별로 사용할 수 있는 특성은 다르다
    • 속성 작성 스타일 가이드
    • 속성을 통해 태그의 부가적인 정보를 설정할 수 있음
    • 요소는 속성을 가질 수 있으며, 경로나 크기와 같은 추가적인 정보를 제공
    • 요소의 시작 태그에 작성하며 보통 이름과 값이 하나의 쌍으로 존재
    • 태그와 상관없이 사용 가능한 속성(HTML Global Attribute)들도 있음
    • HTML Global Attribute
      • 모든 HTML 요소가 공통으로 사용할 수 있는 대표적인 속성 (몇몇 요소에는 아무 효과가 없을 수 있음)
        • id : 문서 전체에서 유일한 고유 식별자 지정
        • class : 공백으로 구분된 해당 요소의 클래스의 목록(CSS, JS에서 요소를 선택하거나 접근)
        • data-* : 페이지에 개인 사용자 정의 데이터를 저장하기 위해 사용
        • style : inline 스타일
        • title : 요소에 대한 추가 정보
        • tabindex : 요소의 탭 순서
  • 시맨틱 태그

    시맨틱 태그

    • HTML5에서 의미론적 요소를 담은 태그의 등장
      • 기존 영역을 의미하는 div 태그를 대체하여 사용
    • 대표적인 태그 목록
      • header : 문서 전체나 섹션의 헤더(머리말 부분)
      • nav : 내비게이션
      • aside : 사이드에 위치한 공간, 메인 콘텐츠와 관련성이 적은 콘텐츠
      • section : 문서의 일반적인 구분, 컨텐츠의 그룹을 표현
      • article : 문서, 페이지, 사이트 안에서 독립적으로 구분되는 영역
      • footer : 문서 전체나 섹션의 푸터(마지막 부분)
    • None semantic 요소는 div, span 등이 있으며 h1, table 태그들도 시맨틱 태그로 볼 수 있음
    • 개발자 및 사용자 뿐만 아니라 검색엔진 등에 의미 있는 정보의 그룹을 태그로 표현
    • 단순히 구역을 나누는 것 뿐만 아니라 '의미'를 가지는 태그들을 활용하기 위한 노력
    • 요소의 의미가 명확해지기 때문에 코드의 가독성을 높이고 유지보수를 쉽게 함
    • 검색엔진최적화(SEO)를 위해서 메타태그, 시맨틱 태그 등을 통한 마크업을 효과적으로 활용해야 함

    form

    • < form > 은 정보(데이터)를 서버에 제출하기 위한 영역
    • < form > 기본 속성
      • action: form을 처리할 서버의 URL
      • method: form을 제출할 때 사용할 HTTP 메서드 (GET 혹은 POST)
      • enctype : method가 post인 경우 데이터의 유형
        • application/x-www-form-urlencoded : 기본값
        • multipart/form-data : 파일 전송시 (input type이 file인 경우)
        • text/plain : HTML5 디버깅 용 (잘 사용되지 않음)

      • value : form contrl에 적용되는 값 (이름/값 페어로 전송됨)

    • input 유형 - 일반
      • 일반적으로 입력을 받기 위하여 제공되며 타입별로 HTML 기본 검증 혹은 추가 속성을 활용할 수 있음

        • hidden : 사용자에게 보이지 않는 input

JSP&Servlet
  • JSP 개념

    JSP(JavaServer Pages)는 서버 측 스크립팅 언어로, 동적 웹 페이지 생성을 위해 사용됩니다. JSP는 HTML 코드와 Java 코드를 조합하여 웹 페이지를 동적으로 생성하고 표현할 수 있습니다. 이를 통해 개발자는 사용자의 요청에 따라 데이터 처리와 로직 수행을 할 수 있습니다. JSP는 웹 애플리케이션 개발에서 주로 사용되며, 서버에서 실행되어 클라이언트에게 동적으로 생성된 HTML을 전달합니다. JSP 파일은 일반적으로 .jsp 확장자를 가지며, 웹 서버에서 실행되어 동적 HTML을 생성합니다. JSP에서는 <% %> 태그를 사용하여 Java 코드를 삽입할 수 있습니다. 이를 통해 개발자는 데이터 처리, 로직 수행, 조건문, 반복문 등을 쉽게 구현할 수 있습니다. JSP는 또한 표현 언어(EL)와 JSTL(JSP Standard Tag Library)을 지원하여 데이터의 출력과 다양한 작업을 간편하게 처리할 수 있습니다. JSP는 다른 서블릿 기술과 함께 사용될 수 있으며, 서블릿 컨테이너에서 실행됩니다. JSP 파일은 서블릿으로 변환되어 컴파일되고, 컨테이너에서 실행됩니다. 이를 통해 JSP는 웹 개발에서 유연하고 효율적인 동적 웹 페이지 생성을 가능케 합니다.

  • Servlet 개념

    서블릿은 자바를 기반으로 하는 서버 측 프로그래밍 기술로, 웹 애플리케이션에서 동적인 컨텐츠를 생성하고 제공하는 역할을 합니다. 서블릿은 웹 서버에서 실행되며, 클라이언트의 요청에 따라 동적으로 HTML, XML, JSON 등의 데이터를 생성하여 전송합니다.

    서블릿은 자바의 표준 API(javax.servlet 패키지)를 구현하여 개발되며, 웹 컨테이너(예: Apache Tomcat)에서 실행됩니다. 서블릿은 생명 주기(Life Cycle)를 가지며, 초기화, 서비스 처리, 소멸 등의 단계를 거쳐 동작합니다.

    서블릿은 HTTP 요청과 응답을 처리하기 위해 HttpServletRequest와 HttpServletResponse 객체를 사용합니다. HttpServletRequest는 클라이언트의 요청 정보를 담고 있으며, HttpServletResponse는 서버의 응답을 생성하여 클라이언트에게 전송합니다.

    서블릿은 자바 코드 안에 HTML이나 다른 웹 요소를 삽입하여 동적인 웹 페이지를 생성할 수 있습니다. 또한, 서블릿은 스레드 기반으로 동작하므로 동시 다중 요청에 대한 처리가 가능합니다.

    이러한 서블릿의 기본 개념을 이해하고 활용하면, 웹 애플리케이션 개발에서 동적인 컨텐츠 생성과 클라이언트 요청 처리에 유용한 서버 측 프로그래밍 기술을 사용할 수 있습니다.

  • HttpServletRequest와 HttpServletResponse

    HttpServletRequest는 웹 애플리케이션에서 클라이언트의 요청 정보를 담고 있는 객체입니다. 서블릿 컨테이너가 클라이언트로부터 받은 HTTP 요청을 분석하여 HttpServletRequest 객체를 생성하고, 서블릿에게 전달합니다. 이 객체를 통해 클라이언트의 요청 메서드(GET, POST 등), URL, 헤더 정보, 파라미터 값 등을 확인할 수 있습니다.

    HttpServletResponse는 서블릿이 클라이언트에게 응답을 전송하는 데 사용되는 객체입니다. 서블릿은 HttpServletResponse 객체를 통해 HTTP 응답 코드, 헤더, 본문 등을 설정하여 클라이언트에게 전송합니다. 이를 통해 서버는 동적으로 생성된 컨텐츠를 클라이언트에게 제공할 수 있습니다.

    HttpServletRequest와 HttpServletResponse는 서블릿 컨테이너에 의해 자동으로 생성되며, 서블릿의 service() 메서드에 인자로 전달됩니다. 서블릿은 이 객체들을 활용하여 클라이언트의 요청을 처리하고, 동적인 컨텐츠를 생성하여 HttpServletResponse 객체에 담아 클라이언트에게 전송합니다.

    HttpServletRequest 객체는 주로 클라이언트의 데이터를 받아오는 역할을 담당하고, HttpServletResponse 객체는 클라이언트에게 응답을 보내는 역할을 담당합니다. 이를 통해 서블릿은 클라이언트와의 상호작용을 가능하게 하며, 웹 애플리케이션에서 동적인 컨텐츠 생성과 데이터 처리를 수행할 수 있습니다.

    요약하자면, HttpServletRequest는 클라이언트의 요청 정보를 담고, HttpServletResponse는 서블릿이 클라이언트에게 응답을 전송하는 데 사용되는 객체입니다. 이를 통해 서블릿은 클라이언트의 요청을 처리하고 동적인 컨텐츠를 생성하여 클라이언트에게 전송할 수 있습니다.

  • JSP 문법
    문법설명
    선언문JSP 페이지 내에서 변수, 메서드, 클래스를 선언합니다.
    <%! %>예: <%! int count = 0; %>
    스크립트릿JSP 페이지에서 Java 코드를 작성합니다.
    <% %>예: <% int result = num1 + num2; %>
    표현식JSP 페이지에서 변수나 연산 결과를 출력합니다.
    <%= %>예: <%= name %>
    주석JSP 페이지에서 주석을 작성합니다.
    <%-- --%>예: <%-- This is a comment --%>
    지시자JSP 페이지의 전체적인 설정 정보를 지정합니다.
    <%@ %>예: <%@ page language="java" contentType="text/html" %>
    include 지시자다른 JSP 파일을 현재 JSP 페이지에 포함시킵니다.
    <%@ include file="include.jsp" %>
    지시문JSP 페이지의 특정 작업을 수행합니다.
    <%@ %>예: <%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
  • EL(Expression Language) 문법

Chapter01 데이터베이스 개요
  • Database란?

    다양한 정보들을 수집, 처리하고, 분석, 응용하는 것은 이제 사회 어느 곳에서나 꼭 필요한 요소가 되었다. 넓은 의미에서의 데이터베이스는 이러한 일상적인 정보들을 모아 놓은 것 자체를 의미한다. 그러나 일반적으로 데이터베이스라고 말할 때는 특정 기업이나 조직 또는 개인이 필요에 의해 논리적으로 연관된 데이터를 모아 일정한 형태로 저장해 놓은 것을 말한다.

  • DBMS(Database Management System)이란?

    많은 사용자들은 보다 효율적인 데이터의 관리 뿐만 아니라 예기치 못한 사건으로 인한 데이터의 손상을 피하고, 필요시 필요한 데이터를 복구하기 위한 강력한 기능의 소프트웨어를 필요로 하게 되었고 이러한 기본적인 요구사항을 만족시켜주는 시스템을 말한다.

  • 데이터 베이스의 주요 특징 4가지
    • 실시간 접근성(real time accessibility)

      데이터베이스는 실시간으로 서비스된다. 사용자가 데이터를 요청하면 몇 시간이나 몇 일 뒤에 결과를 전송하는 것이 아니라 수 초 내에 결과를 서비스한다.

    • 계속적인 변화(continuous change)

      데이터베이스에 저장된 내용은 어느 한 순간의 상태를 나타내지만, 데이터 값은 시간에 따라 항상 바뀐다. 데이터베이스는 삽입(insert), 삭제(delete), 수정(update) 등의 작업을 통하여 바뀐 데이터 값을 저장한다.

    • 동시 공유(concurrent sharing)

      데이터베이스는 서로 다른 업무 또는 여러 사용자에게 동시에 공유된다. 동시(concurrent)는 병행이라고도 하며, 데이터베이스에 접근하는 프로그램이 여러 개 있다는 의미이다.

    • 내용에 따른 참조(reference by content)

      데이터베이스에 저장된 데이터는 데이터의 물리적인 위치가 아니라 데이터 값에 따라 참조된다.

  • DBMS(Database Management System)의 특징
    • 인증된 사용자만이 참조할 수 있도록 보안 기능을 제공한다.
    • 시스템의 갑작스러운 장애로부터 사용자가 입력, 수정, 삭제하던 데이터가 제대로 반영될 수 있도록 보장해주는 기능을 제공한다.
    • 시스템 다운, 재해 등의 상황에서도 데이터를 복구할 수 있는 기능을 제공한다.
  • DBMS(Database Management System)의 장점
    1. 데이터를 통합하여 관리하여 데이터의 중복관리를 최소화한다.
    1. 한번 저장된 데이터를 여러 가지 형태로 표현 가능하도록 한다.
    1. 데이터 일관성을 유지한다.

      중복 저장을 피함으로써 데이터 불일치나 모순성을 해결한다.

    1. 데이터 무결성을 보장한다.

      입력이나 수정 시 다양한 무결성 처리를 자동으로 검증할 수 있다.

    1. 사용의 편의성이 증가되었다

      보다 많은 부분을 DBMS가 지원함으로써 데이터 수집, 분석, 처리가 용이하다.

    1. 데이터 보안이 향상되었다.

      중앙집중식 관리로 일률적인 보안관리가 가능하다.

  • SQL(Structured Query Language)이란?

    관계형 데이터베이스에서 데이터 정의, 데이터 조작, 데이터 제어를 하기 위해 사용하는 언어이다.

  • SQL의 특징
    1. 배우고 사용하기 쉬운 언어이다.
    1. 절차적인 언어가 아니라 선언적인 언어이다.
    1. DBMS의 종류에 얽매이지 않고 사용할 수 있는 장점이 있다.
  • SQL의 종류
    • Query
      • SELECT: 데이터베이스에서 데이터를 검색할 때 사용한다.
    • DML(Data Manipulation Language)
      • INSERT: 테이블에서 새 행을 입력한다.
      • UPDATE: 기존 행 변경한다.
      • DELETE: 행을 제거한다.
      • MERGE: 데이터가 테이블에 존재하지 않으면 INSERT, 존재하면 UPDATE를 수행한다.
    • DDL(Data Definition Language)
      • CREATE: 테이블 등 데이터 구조(객체)를 생성한다.
      • ALTER: 객체를 수정할 때 사용한다.
      • DROP: 객체 제거 시 사용한다.
      • RENAME: 객체 이름을 변경할 때 사용한다.
      • TRUNCATE: 객체 내의 모든 행을 삭제한다.
    • DCL(Data Control Language)
      • GRANT: 데이터베이스 및 해당 구조에 대한 액세스 권한을 부여한다.
      • REVOKE: 부여한 권한을 제거한다.
    • TCL(Transaction Control Language)
      • COMMIT: 변경된 데이터를 테이블에 영구적으로 반영한다.
      • ROLLBACK: COMMIT 이전의 변경사항을 취소한다.
      • SAVEPOINT: ROLLBACK을 할 때 SAVEPOINT로 지정된 일부만 ROLLBACK 한다.

Chapter02 기본 Select문
  • SELECT 문의 기능
    1. 프로젝션(Projection)

      : SQL의 프로젝션 기능으르 사용하면 데이블에서 질의 결과로 반환될 칼럼을 사용자가 필요한 만큼 선택할 수 있다.

    1. 선택(Selection)

      : SQL의 선택기능을 사용하면 테이블에서 질의 결과로 반환될 행을 선택할 수 있으며, 다양한 조건을 사용하여 행을 선택적으로 제한 할 수 있다.

    1. 조인(Join)

      : SQL의 조인 기능을 사용하면 서로 다른 테이블 간에 연결을 생성하여 각 테이블에 저장된 데이터를 함께 가져올 수 있다,

  • SELECT 구문 형식
    SELECT [DISTINCT] { * | column명 | 표현식[별칭] }
  FROM 테이블이름(들)
[WHERE 검색조건(들)]
[GROUP BY 속성이름]
[HAVING 검색조건(들)]
[ORDER BY 속성이름 [ASC|DESC]]
  • 기본 SELECT 구문 특징
    • SELECT 키워드 다음에 별표(*)를 사용하면 테이블에 있는 모든 칼럼을 표시할 수 있다.
    • SELECT 다음에 모든 칼럼명을 나열하여 모든 칼럼을 표시할 수 있다.
    • 조회할 대상(칼럼)을 쉼표로 구분하여 지정하면 테이블의 특정 칼럼을 표시할 수 있다.
    • SELECT절에는 테이블 생성시 정의된 순서와 관계없이 칼럼을 표시할 순서대로 칼럼을 지정한다.
  • 산술 표현식
    • 산술 연산자 종류
      • 더하기(+), 빼기(-), 곱하기(*), 나누기(/)
    • 여러 개의 산술 연산자가 사용되면 *, / 연산자가 +, - 연산자 보다 우선순위가 높다.
    • 같은 우선순위의 연산자는 좌측에서 우측으로 우선순위를 가지게 된다.
    • () 괄호를 사용하게 되면 괄호가 첫 번째 우선순위가 된다.
    • SQL문에서는 FROM 절을 제외한 모든 절에서 산술 연산자를 사용할 수 있다.
    • 산술 연산자 앞 뒤의 공백은 무시한다.
  • NULL 값이란?
    • 한 행의 특정 칼럼에 데이터 값이 지정되지 않았으면 그 값을 널이라고 한다.
    • 널 값은 알 수 없는 값으로서 0 또는 공백과 다르다.
    • 널 값을 포함하는 연산의 경우 결과 값도 널이다.
    • 숫자를 0으로 나누면 오류가 발생하지만 널로 나누면 결과는 널이다.
  • 컬럼 별칭
    • 질의의 결과를 표시할 때 일반적으로 선택한 칼럼의 이름을 열머리글(HEADING)로 사용하는데, 열별칭을 사용하여 열머리글을 변경할 수 있다.
    • 기본적으로 별칭 머리글은 대문자로 표시된다.
    • 별칭에 공백 또는 특수문자(#, $ 등)가 있거나 대소문자를 구분할 경우 별칭을 큰따옴표(” “)로 묶어야 한다.
  • Literal 문자열이란?
    • Literal은 열 이름이나 열 별칭이 아니면서 SELECT 목록에 포함된 문자, 숫자 또는 날짜 등의 상수이다.
    • 리터럴 문자열은 질의 결과에 포함되어 SELECT 목록의 열과 동일하게 취급된다.
    • 날짜 및 문자 리터럴은 반드시 작은 따옴표(’’)로 뭈어야 하지만 숫자 리터럴은 작은 따옴표로 묶지 않아도 된다.
    • 각 문자열은 각 행(row)이 반환될 때마다 한 번씩 출력된다.
  • 중복 행 제거(DISTINCT)란?
    • DISTINCT 키워드를 사용하면 해당 칼럼에 같은 값이 존재하는 경우 결과를 출력할 때 중복되는 값은 하나만 출력해준다.
    • DISTINCT 키워드를 사용하지 않으면 default가 ALL이다. DISTINCT가 중복되는 값을 제거해준다면 ALL 키워드는 모든 값을 출력한다.
    • DISTINCT 다음에 여러 칼럼을 지정할 경우 모든 칼럼에 영향을 준다.
    • SELECT절에서 SELECT 키워드 바로 다음에 DISTINCT 키워드를 넣어야 한다.
  • 행의 제한(WHERE)란?
    • WHERE절을 사용하여 질의에서 반환되는 행을 제한할 수 있다.
    • WHERE절은 만족해야 할 조건을 기술하며 FROM절 바로 다음에 사용한다.
    • 조건을 만족하는 행(조건이 참일 경우)이 반환된다.
    • 문자형 타입과 DATE 날짜 타입을 가진 칼럼을 특정 값과 비교하기 위해서는 인용부호로 묶어서 비교 처리를 해야 하지만, 숫자형 형태의 값은 인용부호를 사용하지 않고 그대로 사용하면 된다.
    • WHERE절에서는 별칭을 사용할 수 없다.
  • 비교 연산자란?
    • 비교 조건은 표현식을 다른 값이나 표현식과 비교하는 조건문에서 사용되면 WHERE절에서 다음 형식으로 사용된다.
    • 종류
      • = (같음), > (보다 큼), >= (크거나 같음), < (보다 작음), <= (작거나 같음), <>, !=, ^=(같지않음)
      • BETWEEN A값 AND B값 : A값과 B값 사이에 포함되는지 비교한다.
      • IN (목록) : 여러 값 중에서 어느 하나와 일치하는지 비교한다. IN은 =ANY, =SOME과 같은 연산을 한다.
      • LIKE :특정 패턴에 속하는 값을 조회하고자 하는 경우
        • Wildcard를 이용한 문자열 부분 매칭
        • ‘ESCAPE’ 뒤에 문자열로 시작하는 문자는 Wildcard가 아닌 것으로 해석한다.
      • IS NULL : 널 여부를 확인한다.
    • 예시

      예제 1) EMP 테이블에서 급여가 1300에서 1700 사이에 해당하는 사원의 성명, 담당업무, 급여, 부서 번호를 조회한다.

      SELECT ename, job, sal, deptno
  FROM emp
 WHERE sal BETWEEN 1300 AND 1700;

      예시 2) EMP 테이블에서 사원번호가 7902, 7788, 7566인 사원의 사원번호, 성명, 담당업무, 급여, 입사일자를 조회한다.

      SELECT empno, ename, job, sal, hiredate
  FROM emp
 WHERE empno IN (7902, 7788, 7566);

      예시 3) EMP 테이블에서 이름의 두 번째 글자가 ‘A’인 사원이름, 급여, 업무를 조회한다.

      SELECT ename, sal, job
  FROM emp
 WHERE ename LIKE '_A%';

      예시 4) player 테이블에서 영문이름에 % 문자가 들어있는 선수정보를 조회한다.

      SELECT player_name 선수이름, e_player_name 영어이름, height 키
  FROM player
 WHERE e_player_name LIKE '%#%%' ESCAPE '#';

      예시 5) EMP 테이블에서 보너스가 NULL인 사원정보를 조회한다.

      SELECT empno, ename, job, sal, comm, deptno
  FROM emp
 WHERE comm IS NULL;
  • 논리 연산자란?
    • WHERE 절 여러 개의 조건을 지정할 때 사용한다.
    • 논리 연산자는 두 조건의 결과를 결합하여 하나의 결과를 생성하거나 단일 조건의 결과를 부정하기도 한다. 조건의 전체가 참인 경우에만 행이 반환된다.
    • 우선순위는 AND, OR, NOT 순이다.
      • AND : 양쪽 조건이 모두 TRUE이면 TRUE를 반환
      • OR : 양쪽 조건이 중 하나만 TRUE이면 TRUE를 반환
      • NOT : 뒤따르는 조건이 False이면 TRUE를 반환
    • ( )는 모든 우선 순위 규칙보다 우선이다.
  • 데이터 정렬의 특징
    • ORDER BY 절은 조회된 데이터들을 다양한 목적에 맞게 특정 칼럼을 기준으로 정렬하여 출력하는 데 사용된다.
    • ORDER BY 절에 칼럼명 대신 SELECT 절에서 사용한 ALIAS 명이나 칼럼 순서를 나타내는 정수도 사용 가능하다.
    • 별도로 정렬방식을 지정하지 않으면 기본적으로 오름차순이 적용되며, 오름차순 정렬 시 숫자 값은 작은 값부터, 날짜 값은 과거 값부터, 문자 값은 영문자 순으로 표시된다.
    • 내림차순(DESCENDING) 정렬 수행 시 DESC 키워드를 지정해야 한다.
    • ORDER BY 절은 항상 SELECT 문의 마지막에 사용한다.

Chapter03 단일 행 함수
  • 단일 행 함수란?
    • 각 행을 각각 일정한 형태로 변환하여 결과를 돌려주는 함수를 말한다. 함수의 인자 값으로 상수, 칼럼값, 변수, 표현식을 사용할 수 있다. 단일 행 함수의 인자는 사용자 지정 상수, 변수 값, 칼럼 이름, 표현식이 있다.
  • 단일 행 함수 특징
    • 데이터 조작을 위해 사용한다.
    • 인자(Argument)를 받아들여 하나의 결과를 반환한다.
    • SELECT, WHERE, ORDER BY 절에 사용할 수 있다.
    • 데이터 타입을 변경할 수 있다.
    • 중첩사용이 가능하다.
  • 단일 행 함수의 종류

    : 적용되는 데이터 타입에 따라 단일 행 함수의 종류를 구분한다.

    • 문자함수 : 문자를 입력하면 문자 또는 숫자 값을 반환한다.
    • 숫자함수 : 숫자를 입력하면 숫자 값을 반환한다.
    • 날짜함수 : 날짜 유형의 값을 연산한다.
    • 변환함수 : 데이터 유형을 변환 한다.
    • 기타함수 : 타입에 상관없이 적용 가능하다.
  • 문자함수 종류
    • LOWER/UPPER : 대상 문자열을 모두 소문자/대문자로 변환
    • INITCAP(char) : 문자열 단어의 첫문자는 대문자 나머지는 소문자로 변경
    • CONCAT(s1, s2) : 두 문자열을 연결
    • SUBSTR(s, m, n) : 문자열 중 지정한 위치에서 지정한 길이만큼의 문자열을 추출
    • INSTR(s1, s2, m, n) : 문자열 내의 특정 문자열의 위치를 숫자로 표시
    • LENGTH(s) : 문자의 길이를 숫자 값으로 표시
    • CHR(n) : ASCII 코드값이 n인 문자 반환
    • ASCII(s): S문자의 ASCII 코드값 반환
    • TRIM: 앞, 뒤에 나오는 특정 문자를 제거
    • TRANSLATE(s, from, to) : 첫 문자를 탐색집합의 첫 문자로 대체(2 번째도 동일)
    • REPLACE(s, p, r) : 특정문자열을 다른 문자열로 대체
  • 숫자 함수 종류
    • ROUND : 지정된 자리로 반올림 수행
    • TRUNC : 지정된 자리에서 버림 수행
    • MOD(M, N) : M을 N으로 나눈 나머지
    • ABS : 숫자 값을 절대값으로 바꿈
    • FLOOR : 해당 수보다 작거나 같은 정수 중 가장 큰 정수 값을 리턴
    • CEIL : 해당 수보다 크거나 같은 정수 중 가장 작은 정수 값을 리턴
    • SIGN : 숫자가 양수이면 1, 음수이면 -1, 0이면 0을 반환
    • POWER(M,N) : m의 n승
  • 날짜 함수 종류
    • MONTHS_BETWEEN(D1, D2) : D1, D2 두 날자 간의 경과 개월 수
      • D1, D2가 같은 날자가 올 경우에는 0을 반환
    • ADD_MONTHS(D1, N) : D1 날짜에 N개월 더함
    • NEXT_DAY(D1, ‘CHAR’) : D1보다 이후 날짜며 지정한 요일에 해당되는 날짜
    • LAST_DAY : 해당 월의 마지막 날짜를 리턴
    • ROUND : 날짜를 년, 월 단위로 반올림
    • TRUNC : 날짜를 년, 월 단위로 버림
  • 변환함수란?

    : 특정 데이터 타입을 다양한 형식으로 출력하고 싶을 경우에 사용되는 함수를 말한다.

    • 종류
      • TO_CHAR(숫자, format)
      • TO_CHAR(날짜, format)
      • 예시

        예제 1) EMP 테이블에서 20번 부서 중 급여를 3자리마다 ‘ , ’를 출력

        SELECT empno, ename, job, sal, TO_CHAR(sal, '999,999')
  FROM emp
 WHERE deptno = 20
 ORDER BY sal desc;

        예제 2) EMP 테이블에서 20번 부서 중 입사 일자를 ‘1998년 1월 1일’의 형태로 출력

        SELECT ename, hiredate, TO_CHAR(hiredate, 'YYYY"년" MM"월" DD"일") t_kor
  FROM emp
 WHERE deptno = 20
 ORDER BY hiredate desc;
  • 일반 함수 종류
    • NVL(column, exp1) : NVL함수는 NULL값을 다른 값으로 바꿀 때 쓰임
    • NVL2(exp1, exp2, exp3) : <expr1>이 null이 아닌 경우에는 <expr2>를 리턴하고, <expr1>이 null일 경우에는 <expr3>을 반환
    • NULLIF(exp1, exp2) : <expr1≥<expr2>일 경우 NULL을, 그렇지 않을 경우, <exp1r>을 반환
    • COALESCE(exp1,,,expN) : exp1, exp2,…expN 중 첫 번째 NULL이 아닌 값을 반환
    • GREATEST(exp1,,,expN) : 하나 이상의 인수 중에서 가장 큰 값을 반환
    • LEAST(exp1, expN) : 인수 Expr의 리스트 중에서 가장 작은 값을 반환
    • DECODE(VALUE, IF1, THEN1, IF2, THEN2…) : IF문이 수행할 수 있는 기능 포함. 만약 일치하는 것을 발견하지 못하면, default값을 반환한다. 만약 default가 생략되면 오라클은 null을 반환
    • CASE 함수 형식
      CASE WHEN 비교조건1 THEN 처리1
     WHEN 비교조건2 THEN 처리2
     ...
     ELSE 디폴트처리
 END
    • 예시

      예제 1) EMP 테이블의 30번 부서원 아닌 사원들의 사원이름, 부서번호, 매니저 번호, 매니저번호가 null이면 ‘상위관리자’로 표시하고, 매니저번호가 있으면 ‘담당’으로 표시하라

      SELECT ename, deptno, mgr, NVL2(mgr, mgr||'담당', '상위자')
  FROM emp
 WHERE deptno <> 30;

      예제 2) EMP 테이블의 30번 부서원들의 사원이름, 업무, 업무가 ‘SALESMAN’인 경우 NULL값으로 변경해서 조회하라.

      SELECT ename, job, NULLIF(job, 'SALESMAN') AS result
  FROM emp
 WHERE deptno = 30;

      예제 3) EMP 테이블에서 급여가 1000보다 작으면 ‘A’, 1000이상 2500미만이면 ‘B’, 2500이상이면 ‘C’로 표시하라

      • decode 함수 이용
      SELECT ename, sal,
       DECODE(SIGN(sal-1000), -1, 'A', DECODE(SIGN(sal-2500), 01, 'B', 'C')) grade
  FROM emp;
      • case 함수 이용
      SELECT ename, sal,
       CASE WHEN sal < 1000 THEN 'A'
            WHEN sal < 2500 THEN ' B;
            ELSE grade
  FROM emp;
