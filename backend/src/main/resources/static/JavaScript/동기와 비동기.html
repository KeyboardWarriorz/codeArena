동기(Synchronous)

동기적인 코드 실행은 코드가 순차적으로 실행되는 것을 의미합니다. 즉, 한 작업이 완료되기 전에 다음 작업이 실행되지 않습니다. 동기 코드는 코드의 실행 순서를 명확하게 파악할 수 있으며, 순차적인 로직을 구현하는 데 적합합니다.

console.log('첫 번째');
        console.log('두 번째');
        console.log('세 번째');

▲ 위의 코드는 순차적으로 실행되며, "첫 번째" -> "두 번째" -> "세 번째" 순서대로 출력됩니다.

비동기(Asynchronous)

비동기적인 코드 실행은 코드가 순차적으로 실행되지 않고, 작업이 완료되지 않은 상태에서 다음 작업이 실행될 수 있는 것을 의미합니다. 비동기 코드는 네트워크 요청, 파일 읽기, 타이머 등의 작업에서 주로 사용됩니다. 이러한 작업은 일반적으로 시간이 걸리기 때문에, 비동기 방식을 사용하면 브라우저의 응답성을 유지하면서 작업을 처리할 수 있습니다.

console.log('첫 번째');
        setTimeout(function() {
            console.log('두 번째');
        }, 1000);
        console.log('세 번째');

▲ 위의 코드에서 setTimeout 함수는 1초 후에 콜백 함수를 비동기적으로 실행합니다. 따라서 "첫 번째"와 "세 번째"는 즉시 출력되고, "두 번째"는 1초 후에 출력됩니다.

👩‍🏫

비동기 코드를 처리하기 위해 자바스크립트에서는 콜백 함수, 프로미스, async/await 등의 패턴과 API를 제공합니다. 이러한 기능을 사용하여 비동기 작업을 효율적으로 처리하고, 코드의 가독성과 유지보수성을 높일 수 있습니다.
