데이터베이스 트랜잭션은 데이터베이스에서 수행되는 논리적인 작업의 단위를 나타냅니다. 트랜잭션은 하나 이상의 데이터베이스 작업(쿼리, 업데이트 등)을 묶어서 한꺼번에 실행하며, 모든 작업이 성공적으로 수행되어야 데이터베이스에 영구적인 변경이 발생합니다. 트랜잭션은 데이터의 일관성과 무결성을 보장하고, 장애 발생 시 데이터의 원자성을 유지하기 위해 사용됩니다.

트랜잭션은 ACID 원칙을 따릅니다. ACID는 원자성(Atomicity), 일관성(Consistency), 고립성(Isolation), 지속성(Durability)의 약자입니다.

특성설명
원자성 (Atomicity)트랜잭션은 모든 작업이 전부 실행되거나 전혀 실행되지 않는 원칙을 따릅니다.
일관성 (Consistency)트랜잭션이 실행 전후에 데이터베이스가 일관된 상태를 유지하는 원칙을 의미합니다.
고립성 (Isolation)여러 트랜잭션이 동시에 실행될 때 각각의 트랜잭션이 다른 트랜잭션에 영향을 주지 않도록 보장합니다.
지속성 (Durability)트랜잭션이 성공적으로 완료된 후에도 그 결과가 영구적으로 유지되는 원칙을 의미합니다.

트랜잭션은 데이터베이스 시스템에서 핵심적인 개념으로, 여러 개의 데이터베이스 작업을 논리적인 단위로 묶어서 일관성과 안정성을 유지합니다. 트랜잭션은 장애나 충돌 상황에서 데이터의 일관성을 보장하고, 데이터베이스의 신뢰성과 안정성을 제공합니다. 또한, 트랜잭션을 사용하여 데이터베이스의 동시성 제어와 병행성을 관리할 수 있습니다.
