트랜잭션(Transaction)은 데이터베이스에서 수행되는 작업의 논리적인 단위입니다. 여러 개의 데이터베이스 조작(쿼리, 삽입, 수정, 삭제 등)을 하나의 논리적인 단위로 묶어서 실행하며, 모든 작업이 정상적으로 완료되면 데이터베이스의 상태를 변경합니다. 트랜잭션은 ACID(원자성, 일관성, 고립성, 지속성) 원칙을 따라 안전하고 일관성 있는 데이터 처리를 보장합니다.

📒

트랜잭션 ACID 원칙

원자성(Atomicity)

일관성(Consistency)

고립성(Isolation)

지속성(Durability)

👩‍🏫

트랜잭션은 데이터베이스에서 데이터 일관성과 안전한 처리를 위해 필요한 개념입니다. 트랜잭션은 한 번에 하나 이상의 작업을 수행할 수 있으며, 작업이 성공적으로 완료되면 영구적으로 데이터베이스에 반영됩니다. 이를 통해 데이터의 일관성과 무결성을 유지합니다.
